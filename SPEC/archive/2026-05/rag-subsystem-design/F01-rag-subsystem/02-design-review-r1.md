## Axis 1 - Two Proposals and Required Proposal Coverage

## Axis 4 - Dependency Realism

Blocking finding. The core dependency set is mostly realistic: `better-sqlite3`, `sqlite-vec`, current-Node ESM/prebuild verification, and optional `@lancedb/lancedb` fallback are all documented ([saivage/SPEC/2026-05/rag-subsystem-design/F01-rag-subsystem/02-design-r1.md](saivage/SPEC/2026-05/rag-subsystem-design/F01-rag-subsystem/02-design-r1.md#L598-L619), [saivage/SPEC/2026-05/rag-subsystem-design/F01-rag-subsystem/02-design-r1.md](saivage/SPEC/2026-05/rag-subsystem-design/F01-rag-subsystem/02-design-r1.md#L623-L628)). The problem is the ingest lock dependency story: the module layout promises a `proper-lockfile`-equivalent lock with no dependency, the failure table depends on `flock(2)` semantics, and the dependency table claims `node:fs` `flock` is enough while the open questions still ask the implementer to confirm that primitive ([saivage/SPEC/2026-05/rag-subsystem-design/F01-rag-subsystem/02-design-r1.md](saivage/SPEC/2026-05/rag-subsystem-design/F01-rag-subsystem/02-design-r1.md#L47-L50), [saivage/SPEC/2026-05/rag-subsystem-design/F01-rag-subsystem/02-design-r1.md](saivage/SPEC/2026-05/rag-subsystem-design/F01-rag-subsystem/02-design-r1.md#L330-L341), [saivage/SPEC/2026-05/rag-subsystem-design/F01-rag-subsystem/02-design-r1.md](saivage/SPEC/2026-05/rag-subsystem-design/F01-rag-subsystem/02-design-r1.md#L610-L629)). A design that requires cross-process ingest exclusion needs to name an implementable lock primitive or add the dependency.

## Required Changes

1. Move Proposal B's on-disk layout under the required project-local `.saivage/rag/<dataset-or-collection>/` tree, or explicitly reshape the alternative so it still satisfies the topic's required on-disk-layout contract while being rejected for v1.
2. Replace the ambiguous lock strategy with an implementable cross-process locking primitive. Either add a realistic dependency such as `proper-lockfile`, specify an atomic-directory/lockfile algorithm with stale-lock behavior and tests, or name a verified Node 24 API; then update the dependency table, failure mode, open question, and concurrency test surface consistently.
3. Make embedding drift refusal explicitly include `releaseFingerprint` wherever the failure mode is specified, not only in the type/schema/cache examples.
4. Remove `overshoot` from the public `QueryOptions` surface unless the design intentionally wants callers to tune post-filter recall; if it remains public, reconcile that with the analysis and with the design sentence saying callers never see the knob.
5. Fix the remaining real repo path references inside code comments, especially the [saivage/src/config.ts](saivage/src/config.ts) comments in the config snippets, so generated-doc file references obey the markdown-link rule.
