# G32 — APPROVED

**Chosen proposal**: Proposal A (per [02-design-r5.md](02-design-r5.md)) — replace the host `find(1)` subprocess in `search_files` with an in-process bounded async walker built on `node:fs/promises.opendir`, satisfying the MCP no-subprocess contract and the G30 no-sync-fs invariant. A hand-rolled `globToRegExp` provides segment-aware semantics where `**` matches zero-or-more path segments and `*` stays within a single segment; mixed patterns like `foo**bar` are rejected as `INVALID_ARGUMENT`. The walker returns the success envelope `{ files, truncated, truncated_reason, skipped, max_results, max_depth, max_ms }`. Root `opendir(dir)` failures raise a structured `READ_DIRECTORY_FAILED` / `PERMISSION_DENIED` / `NOT_FOUND` / `NOT_A_DIRECTORY` / `IO_ERROR` envelope (no partial success surface). Sub-directory failures are classified via the G31 `classifyFsError` helper into a recoverable `skipped[]` list or a hard error per policy. Empty pattern emits `INVALID_ARGUMENT` at the handler boundary as the single source of truth. Proposal B (shared `fsWalker.ts` module) is rejected as premature abstraction.

Four scoped grep gates (removal of obsolete guard string, presence-in-case of the new INVALID_ARGUMENT literal, absence-in-complement, absence-in-helper) are jointly satisfiable only by the approved design.

**Sequencing**: Hard prereq G30 (async fs baseline + no-sync-fs test) and G31 (imports `parseNonNegativeInt` and `classifyFsError`). Same-file disjoint with G33/G34/G35. Recommended order G30 -> G31 -> G32 -> G34 -> G33 -> G35.

**Daemon impact**: `search_files` becomes deterministic across hosts (no dependency on host `find(1)`) and bounded by results/depth/time caps. Failure modes are structured; previous silent `{ files: [] }` swallowing is gone. Any saivage-v3 restart remains operator-gated.
