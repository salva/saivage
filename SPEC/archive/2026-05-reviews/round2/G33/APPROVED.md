# G33 — APPROVED

**Chosen proposal**: Proposal A (per [02-design-r4.md](02-design-r4.md)) — replace the brittle regex parsing of raw DDG HTML with a DOM-parsed `web_search` that consumes the G34-owned helpers in [src/mcp/httpFetch.ts](../../../../src/mcp/httpFetch.ts). The handler destructures `TimedFetch { response, signal, timedOut, dispose }` from `fetchWithTimeout`, wraps the post-fetch block in `try { ... } finally { timed.dispose(); }`, threads `timed.signal` into `readBoundedTextBody(response, WEB_SEARCH_MAX_BYTES, signal)`, catches the mid-body abort throw and forwards `{ timedOut: timedOut() }` into `classifyNetworkError` to deterministically map it to `code: "TIMEOUT"`, and uses `discardBody` on the `!response.ok` branch instead of an ad-hoc body cancel. Result extraction uses `node-html-parser` (with `he` for entity decoding) over `extractDdgResults`. Single-pass `uddg` decoding via `URLSearchParams.get` validated by `new URL`. Config additions live in a new `webSearch` group with `webSearchMaxResults`, `webSearchTimeoutMs`, `webSearchMaxBytes` placed strictly after `fetchTimeoutMs`. Pinned fixture matrix covers parser smoke, uddg decode, empty query, max_results clamp, markup drift -> `NO_RESULTS_PARSED`, upstream 503 -> `UPSTREAM_HTTP_ERROR`, timeout, oversize stream, ENOTFOUND, plus a timer-cleanup regression. Proposal B (pluggable backend) is rejected as dead code with one implementation.
