# G34 — APPROVED

**Chosen proposal**: Proposal B (per [02-design-r3.md](02-design-r3.md)) — extract a shared HTTP helper module [src/mcp/httpFetch.ts](../../../../src/mcp/httpFetch.ts) exporting `fetchWithTimeout` (returns a `TimedFetch` with `response`, `signal`, `timedOut()`, and `dispose()`), `readBoundedTextBody` and `readBoundedBinaryBody` (both accept an `AbortSignal`, throw `signal.reason` on mid-stream abort, and stream-decode UTF-8 with `{ stream: true }` so truncated multi-byte tails do not leak `\uFFFD`), `discardBody`, and `classifyNetworkError` (with a `timedOut` flag so any abort observed mid-body deterministically maps to `TIMEOUT` regardless of `error.name`). All `fetch_url`, `fetch_page_text`, `downloadUrl`, `download_file`, and `download_with_fallbacks` call sites use the helper inside `try { ... } finally { timed.dispose(); }` blocks so the underlying `setTimeout` handle is cleared on every exit. Header fast-fail (`!response.ok` or `Content-Length > cap`) calls `discardBody` to abort the stream without buffering. `mcp.maxFetchChars` is renamed to `mcp.maxFetchBytes` (no compat shim) and the arg `max_chars` becomes `max_bytes`. `downloadUrl` returns a typed `DownloadOutcome` union carrying `ClassifiedHttpError`, and the propagated codes are `INVALID_ARGUMENT`, `TIMEOUT`, `NETWORK_ERROR`, `UPSTREAM_HTTP_ERROR`, `RESPONSE_TOO_LARGE`, `IO_ERROR`. Proposal A (per-call cap with no shared helper) is rejected because it leaves four call sites with copy-paste streaming code.
