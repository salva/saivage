Reviewer: GitHub Copilot

Round 2 fixes the important architectural problems from round 1: the cop is deleted rather than renamed, stale runtime config is supposed to fail loudly, the source/doc surface is much wider, and the no-cop invariant now includes the previously missing residue classes. I agree with the direction and with the no-fragile-heuristics reading: structural URL, byte, filesystem-containment, and env-scrubbing guards stay; LLM/classifier gates on untrusted content go away.

## Findings

1. **The stale-security regression test is specified against an unexported schema, so the plan is not executable as written.** [src/config.ts](../../../../src/config.ts#L53-L191) defines `configSchema` as a module-local `const`, while [src/config.test.ts](../../../../src/config.test.ts#L1-L124) currently imports only `loadConfig` and `expandHome`. [03-plan-r2.md](03-plan-r2.md#validation) tells the implementer to call `configSchema.safeParse(raw)`, but no edit step exports that symbol or switches the test to the public load path. That leaves the fail-loud stale-config proof underspecified. Fix it by either explicitly exporting `configSchema` as part of E4 and treating that as an intentional test surface, or preferably by writing a project-local .saivage/saivage.json with a `security` key in [src/config.test.ts](../../../../src/config.test.ts) and asserting `loadConfig(true, projectRoot)` throws a Zod `unrecognized_keys` error mentioning `security`. The latter better proves the boot-time behavior required by the no-backward-compatibility rule.

2. **The exact-match inventory still omits one removal site for the `PromptInjection` needle.** The live grep shows `PromptInjection` also appears in [src/server/bootstrap.ts](../../../../src/server/bootstrap.ts#L13-L150) through `createPromptInjectionCop`. [01-analysis-r2.md](01-analysis-r2.md#24-full-residue-vocabulary) maps the bootstrap file under `promptInjectionCop` and `prompt-injection-cop`, and [03-plan-r2.md](03-plan-r2.md#e3--strip-cop-wiring-from-srcserverbootstrapts) removes the import/call, so the implementation would still delete it. But the brief asks for every needle removal site mapped; add bootstrap to the `PromptInjection` row so the inventory and the grep proof align exactly.

## Principle Compliance

- **No regex for parsing user intent:** satisfied. The proposal deletes the LLM cop and its verdict parsing path; remaining regexes are transport/content normalization and secret scrubbing, not intent classifiers.
- **Avoid hardcoded values; prefer config:** satisfied. Cop-only hardcoded scanner defaults disappear; surviving data-tool limits remain runtime `mcp` config knobs.
- **No fragile agent-tool-call heuristics:** satisfied. No model verdict gates `fetch_url`, `download_file`, or `download_with_fallbacks`; those tools return data subject only to structural boundaries.
- **Architecture-first / no backward compatibility:** direction satisfied by `.strict()`, but the test plan must prove it through an executable unit/integration test as noted above.
