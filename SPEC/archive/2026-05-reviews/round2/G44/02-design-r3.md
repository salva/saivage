# G44 — Design r3

Round 3. Round 2 design is at [02-design-r2.md](02-design-r2.md); the
reviewer's findings are in [04-review-r2.md](04-review-r2.md). Round 2
adopted Proposal A (targeted rewrite of channels.md + targeted edits
to agent-chat.md, plus dist rebuild) and the reviewer found no fault
with the proposal shape, edit map (within the round 2 scope), or
wording rules. The only blocker was a verification-gate problem.

The substantive changes in this round are confined to:

1. The verification gate is split into a cross-doc clean check
   (tokens with no legitimate use anywhere in `docs/`) and a
   file-scoped check (tokens that legitimately appear in valid
   prose elsewhere and so are checked only against the specific
   stale files). This makes the gate executable inside the actual
   G44 edit scope.
2. The edit map is extended by one row covering the `ChatLogSchema`
   table cell in
   [docs/internals/data-model.md](../../../../docs/internals/data-model.md#L28),
   which carries the same flat-chat-log-path drift as
   [docs/internals/agent-chat.md](../../../../docs/internals/agent-chat.md#L57-L58)
   per
   [01-analysis-r3.md](01-analysis-r3.md#L41-L52).
3. The token `stop\(\)` is dropped from the gate. In channels.md it
   appears only inside the four-line stale `interface Channel`
   block, which is removed wholesale by the rewrite; the only
   remaining `stop()` in `docs/` is the valid `startServer` return
   type in
   [docs/internals/server.md](../../../../docs/internals/server.md#L48).

Everything else in
[02-design-r2.md](02-design-r2.md) — Proposal A's shape, the rejected
Proposal B and Proposal C, the wording rules for the rewrite, the
non-goals — stands without change.

## Verification gate (replacement for r2 step 3 / step 5)

The single regex from
[03-plan-r2.md](03-plan-r2.md#L221-L229) is replaced by a partitioned
gate. Pattern partitioning is grounded in
[01-analysis-r3.md](01-analysis-r3.md#L73-L121); the partitions are
authoritative — the plan only encodes the commands.

### Cross-doc-clean check

Tokens that have **no legitimate use anywhere in `docs/`**. Checked
tree-wide against `docs/**/*.md` (source) and
`docs/.vitepress/dist/**` (built). Expected: zero matches outside
the SPEC review trail.

Patterns (combined into one alternation for readability):

- `channels/(cli|oneshot|index)\.ts`
- `publish\(event`
- `chat-chunk`
- `Three concrete channel implementations`
- `start\(runtime`

### File-scoped check

Tokens that legitimately appear elsewhere in valid prose. Each
pattern is checked only against the files where it is stale.

| Pattern | Files checked (expected: zero matches) |
| --- | --- |
| `interface Channel\b` | [docs/internals/channels.md](../../../../docs/internals/channels.md) only |
| `One-shot CLI` | [docs/internals/agent-chat.md](../../../../docs/internals/agent-chat.md) only |
| `tmp/chats/<sessionId>\.json` (flat, no `<channel>` segment) | [docs/internals/agent-chat.md](../../../../docs/internals/agent-chat.md) and [docs/internals/data-model.md](../../../../docs/internals/data-model.md) |

Files explicitly **not** checked by the file-scoped gate, because
their occurrences of these tokens are valid prose:

- [docs/internals/server.md](../../../../docs/internals/server.md#L48)
  — `stop()` is the valid `startServer` return type; `stop\(\)` is
  dropped from the gate entirely (the channels.md occurrence is
  removed wholesale via the `interface Channel` rewrite).
- [docs/guide/quickstart.md](../../../../docs/guide/quickstart.md#L64)
  — `One-shot CLI` is a valid Planner mode heading.

The same partitioned check is repeated against the rebuilt dist
tree in step 5 of the plan, with the file-scoped patterns applied
to the corresponding `.html` and JS chunks under
`docs/.vitepress/dist/internals/`.

## Cost / risk (delta)

- One additional table-cell edit to `data-model.md` adds ~1 line of
  net change and one regenerated dist file. Total PR remains a
  small docs PR.
- Risk: the partitioned gate is more nuanced than a single
  cross-doc regex. Mitigated by writing both invocations verbatim
  in the plan (no improvisation) and by listing the expected
  zero-match files explicitly so the implementer cannot accidentally
  scope-creep the check.
- Risk: someone later legitimately adds the phrase `interface
  Channel` to `agent-chat.md` or another file and the file-scoped
  gate misses it. Acceptable in this PR — the broader anti-drift
  lint is G40's level-up; G44 only verifies its own scope.
