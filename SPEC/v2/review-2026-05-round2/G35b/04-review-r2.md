## Blocking Findings

1. Schema-layer full-replacement semantics are still only partially pinned. The predicate layer now has explicit replacement tests for both `credentialLexemes: ["PII"]` and `configPointerSuffixes: []` [SPEC/v2/review-2026-05-round2/G35b/02-design-r2.md](02-design-r2.md#L477-L521), and the integration layer has matching replacement cases [SPEC/v2/review-2026-05-round2/G35b/02-design-r2.md](02-design-r2.md#L595-L616). The schema test plan, however, only checks omitted defaults, invalid values, and the empty suffix-array override [SPEC/v2/review-2026-05-round2/G35b/02-design-r2.md](02-design-r2.md#L546-L566), [SPEC/v2/review-2026-05-round2/G35b/03-plan-r2.md](03-plan-r2.md#L259-L285). That proves suffix replacement for the empty-array case, but it does not independently prove that a supplied non-empty credential list is parsed verbatim instead of being unioned with defaults. Add schema-level assertions such as `credentialLexemes: ["PII"]` resolving exactly to `["PII"]`, and preferably `configPointerSuffixes: ["_BUILDFILE"]` resolving exactly to `["_BUILDFILE"]`, so the config parser itself proves full replacement before the predicate/integration tests get involved.

2. The new sentinel grep gate for the hyphen compiler fix does not run as written. Gate 20 proposes `grep -n '\\\\[_-\\\\]' src/security/secrets.ts` as the lock for the `_` to `[_-]` rewrite [SPEC/v2/review-2026-05-round2/G35b/03-plan-r2.md](03-plan-r2.md#L465-L470), but that pattern is parsed by grep as a malformed character class and fails with `Invalid range end` before it can inspect the file. Gates 17-19 usefully pin the `SOME_API-KEY`, singleton `PII`, and empty-suffix cases [SPEC/v2/review-2026-05-round2/G35b/03-plan-r2.md](03-plan-r2.md#L456-L464), but gate 20 must be changed to a fixed-string check, for example `rg -n -F 'replace(/_/g, "[_-]")' src/security/secrets.ts` plus a fixed-string boundary check for `(?:^|[_-])${escaped}S?(?:$|[_-])`, or `grep -nF '[_-]' src/security/secrets.ts` if the intended gate remains broad.
