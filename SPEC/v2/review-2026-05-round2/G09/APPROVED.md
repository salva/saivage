# G09 — APPROVED

**Chosen proposal**: Design B (per [02-design-r5.md](02-design-r5.md)) — replace the brittle `PLAN_COMPLETE` text marker with a structured `plan_done(reason)` MCP tool call. `PlanService.plan_done` is stateless and typed `Promise<{ ok: true; recorded: boolean } | PlanError>`. A new `detectTerminalToolCall(toolCalls: ToolCallResult[], dispatchResult: DispatchResult)` hook on `BaseAgent` (overridden by `PlannerAgent`) returns the structured completion only when the tool batch is exactly one successful `plan_done` call matched by `toolUseId`. `runLoop` re-checks `this.abortSignal?.aborted || dispatchResult.aborted` after pushing tool results and before any terminal-tool return, so abort precedence is enforced in actual code. The planner returns `{ completion: "plan_done", summary }` and continuous-improvement restart/termination keys on that tagged shape, not on a string-shape `hasSummary` guard. Proposal A (hardening the regex) is explicitly rejected — it preserves the free-text control protocol and contradicts the architecture-first rule.

**Approved by**: GPT-5.5 (copilot) reviewer at round 5 — see [04-review-r5.md](04-review-r5.md). All required changes across r1-r4 addressed (terminal-tool path, explicit PlanService wiring then dropped in favor of stateless service, structural recovery discriminator, union-typed method, MCP dispatch tests, PLAN_COMPLETE grep reconciled via test-side token concatenation, exclusivity + abort precedence, real `McpRuntime.callTool` semantics in the stub, typed `ToolCallResult[]` signature, correct `PlannerAgent.create(ctx, childSpawner, { abortSignal })` test construction, rejected-batch-then-valid-single regression test locking in exclusivity + current-call reason selection).

**Implementation pointer**: [03-plan-r5.md](03-plan-r5.md). Test stub mirrors production `McpRuntime.callTool` semantics: throws on `isError`, returns `content` on success. Negative regression test uses `const LEGACY_TOKEN = "PLAN_" + "COMPLETE"` so the literal never appears in source — `grep -rn PLAN_COMPLETE src/ prompts/` returns zero after the change.

**Daemon impact**: `saivage` (10.0.3.111), `diedrico` (10.0.3.113), `saivage-v3` (10.0.3.112) — all bind-mount the saivage source. Operator-gated `saivage-v3` smoke restart only if necessary; other containers untouched. `saivage-v3-getrich-v2` (10.0.3.170) unaffected.
