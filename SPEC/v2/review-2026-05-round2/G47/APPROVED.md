# G47 — APPROVED

**Chosen proposal**: Proposal A (per [02-design-r3.md](02-design-r3.md)) — targeted in-channel fix for the three coupled defects in src/server/telegram-bot.ts: (1) silent unauthorized drop replaced with an explicit `ctx.reply` denial; (2) `bot.start` await via a readiness-handoff Promise that resolves on grammy's `onStart` and rejects on pre-`onStart` errors (with explicit documentation that grammy invokes `onStart` BEFORE the first `getUpdates` loop, so first-poll/steady-state failures are handled by a post-`onStart` `.catch` log-only arm); (3) persistence schema rewritten from `chatIds: string[]` to `entries: Array<{ chatId, userId, subscribedAt }>` so boot hydration respects `allowedUserIds`.
