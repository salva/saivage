# G26 — APPROVED

**Chosen proposal**: Proposal A (per [02-design-r7.md](02-design-r7.md)) — eliminate the obsolete `model_overrides` legacy routing tier. `ProjectConfigSchema` is wrapped in `z.preprocess` that emits a typed Zod custom issue with `fatal: true` at path `["model_overrides"]` and message `model_overrides is a removed legacy v1 routing field. Delete it from .saivage/config.json and use ProjectConfig.routing.roles instead.` when the legacy key is present in the raw input; otherwise the raw value is forwarded to the inner object schema whose default strip-unknown-keys behavior is preserved (no `.passthrough()`, no `.strict()`, so unrelated unknown keys like `notifications`/`provider` continue to be silently stripped as today). The seeder stub at [src/store/project.ts](../../../../src/store/project.ts) is dropped, and the resolver legacy arm collapses: `resolveLegacyModels` is deleted, `resolvePreferredModels` returns runtime defaults directly, the resolver's source union narrows to `"routing" | "runtime-default"`, and `resolveSource`'s `model_overrides` branch is removed. Six routing-related guide docs under [docs/guide/](../../../../docs/guide/) drop every `model_overrides` reference. Proposal B (also delete `ResolvedModelRoute.source`) is rejected because G25's typed `NoAllowedRouteMatchError` design and future telemetry depend on it.

Any config that grows back a `model_overrides` key will fail loudly at load time with the typed Zod issue. Any saivage-v3 restart remains operator-gated.
